1. Java相关的框架、中间件总结

​ Java之所以这些年屹立不倒始终排在编程排行版的前三位，除了和它自身的优秀特性有关外，还和它强大的社区平台有关，首先为我们所熟之的就是Spring开源组织所带给我们的Spring全家桶，Spring全家桶为我们在web开发的各个领域提供了优秀的框架让我们可以快速的构建我们的应用，所以熟悉Spring全家桶并能准确的说出它们的概念及作用是一个Java从业人员所必须掌握的。当然除了Spring全家桶外，还有众多的开源中间件值得我们去学习和掌握，如Redis、RabbitMQ、Nginx、ElasticSearch等等。

2. Spring全家桶

​ 对于Spring全家桶的概念我的想法是：首先需要知道官方文档是如何介绍的（PS :英语好的可以先看官方的英文介绍，官方的文档介绍通常会很长，所以只需要将核心的部分进行记录即可），然后再根据自己的理解和想法更加细粒度的解析并表达出来。

### 2.1 Spring Framework

Spring makes it easy to create Java enterprise applications. It provides everything you need to embrace the Java language in an enterprise environment,Spring supports a wide range of application scenarios,Spring is open source. It has a large and active community.

​ 以上便是Spring官方文档中对Spring进行介绍的核心内容。我的总结：Spring是一个轻量级、开源的企业级Java应用框架，它所提供的众多特性及强大的社区平台使构建Java应用程序更加的高效、易维护。

2.2 Spring IOC

IOC is also known as dependency injection (DI). It is a process whereby objects define their dependencies, that is, the other objects they work with, only through constructor arguments, arguments to a factory method, or properties that are set on the object instance after it is constructed or returned from a factory method. The container then injects those dependencies when it creates the bean. This process is fundamentally the inverse, hence the name Inversion of Control (IOC), of the bean itself controlling the instantiation or location of its dependencies by using direct construction of classes, or a mechanism such as the Service Locator’s pattern.

​ “控制反转（IOC）”也称为“依赖注入（DI）”，是一个定义对象依赖的过程，对象只和构造参数，工厂方法参数，对象实例属性或工厂方法返回相关。容器在创建这些 bean 的 时候注入这些依赖。对象实例的创建由 其提供的构造方法或服务定位机制来实现。控制反转的理解应该是这样的，原先由我们实例化对象并注入依赖的控制权交由给了IOC容器，而IOC容器又是遵循依赖倒置的原则通过面向抽象编程而非具体实现的方式，优雅的降低了对象间的耦合，为编程带来了极大的便利。

2.3 Spring AOP

Aspect-oriented Programming (AOP) complements Object-oriented Programming (OOP) by providing another way of thinking about program structure. The key unit of modularity in OOP is the class, whereas in AOP the unit of modularity is the aspect. Aspects enable the modularization of concerns (such as transaction management) that cut across multiple types and objects. (Such concerns are often termed “crosscutting” concerns in AOP literature.

​ 我的理解：AOP即面向切面(方面)编程，是对OOP面向对象编程的一种补充。传统的OOP是自上而下编写我们的主业务逻辑代码，而在编写主业务逻辑代码时常常需要补充一些辅助逻辑代码来帮助我们更好的管理应用程序，如日志、事务、权限控制等等，但是这些辅助的业务逻辑往往存在于大量的方法中，这就产生了横切性质的问题。传统的OOP不擅长编写从左到右的业务逻辑将众多需要辅助逻辑的代码进行一次编写，这样就产生了大量冗余的代码。而AOP编程将需要重用的辅助逻辑（横切关注点）通过动态代理的方式织入到众多的方法（核心关注点）中。

2.4 Spring MVC

Spring Web MVC is the original web framework built on the Servlet API and included in the Spring Framework from the very beginning. The formal name “Spring Web MVC” comes from the name of its source module spring-webmvc but it is more commonly known as “Spring MVC”.

​ Spring MVC 其正式的名称应该叫 Spring Web MVC，但我们通常叫它Spring MVC。Spring MVC是基于Servlet API而构建的一个web框架。

2.5 Spring Boot

Spring Boot makes it easy to create stand-alone, production-grade Spring based Applications that you can “just run”.
We take an opinionated view of the Spring platform and third-party libraries so you can get started with minimum fuss. Most Spring Boot applications need very little Spring configuration

​ Spring Boot 是对第三方类库及Spring平台的整合，它能够快速的构建独立的、生产级别的基于Spring的应用程序。它”约定大于配置“的理念简化了原先Spring应用程序的大量配置。当然这和它的启动原理有关。

2.6 Spring Cloud

Spring Cloud provides tools for developers to quickly build some of the common patterns in distributed systems (e.g. configuration management, service discovery, circuit breakers, intelligent routing, micro-proxy, control bus, one-time tokens, global locks, leadership election, distributed sessions, cluster state).

​ Spring Cloud 是一套为开发人员提供分布式系统所需常用组件的工具，如注册中心，分布式配置中心，分布式会话，全局锁，熔断器，只能路由，控制总线，一次性令牌，集群状态等等。

3.中间件

3.1 缓存数据库Redis

Redis 是一个开源（BSD许可）的，内存中的数据结构存储系统，它可以用作数据库、缓存和消息中间件。 它支持多种类型的数据结构，如 字符串（strings）， 散列（hashes）， 列表（lists）， 集合（sets）， 有序集合（sorted sets） 与范围查询， bitmaps， hyperloglogs 和地理空间（geospatial） 索引半径查询。 Redis 内置了 复制（replication），LUA脚本（Lua scripting）， LRU驱动事件（LRU eviction），事务（transactions） 和不同级别的 磁盘持久化（persistence）， 并通过 Redis哨兵（Sentinel）和自动 分区（Cluster）提供高可用性（high availability）。

3.2 消息中间键RabbitMQ

RabbitMQ is a message broker: it accepts and forwards messages. You can think about it as a post office: when you put the mail that you want posting in a post box, you can be sure that Mr. or Ms. Mail-person will eventually deliver the mail to your recipient. In this analogy, RabbitMQ is a post box, a post office and a postman.

​ RabbitMQ就是一个消息代理商。它接受并转发消息。我们知道消息中间件通常能带来这三大好处：解耦、异步、削峰。以上将RabbitMQ描述为邮局其实就是描述了消息中间件的两大好处：解耦、异步。
